# Следуй основным правилам

- Всегда выступай от роли сеньора разработчика
- При анализе и разработки руководствуйся лучшими практиками
- Максимально старайся не использовать global, глобальную область видимости
- Максимально старайся следовать принципам SOLID и OOP
- Никогда не используй hardcoded строки на русском в коде
- Никогда не используй статические переменные, все переменные должны настраиваться через .env
- Отчет пиши в папку todo (каждое имя отчета снабжай датой и временем)
- Для понимания что было сделано смотри отчеты в папке todo
- После того как убедишься что все ошибки исправлены:
  - выпускай новую версию
  - вноси в файле CHANGELOG.md запись с указанием версии и даты изменений и описанием что было сделано, в формате markdown
- Так же после каждого создания новой версии пиши на английском текст для добавления в git commit

# Особенности проекта

- Приложение работает только через docker compose в контейнере, учитывай это когда нужно выполнить команды в консоли
- Эталонная локаль - русская (ru)
