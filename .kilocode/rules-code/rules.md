# Правила проведения работ

## СИСТЕМА УПРАВЛЕНИЯ СОСТОЯНИЕМ (Persistence)

Ты ведешь лог в папке `./.refactoring/`:

1. `state.md`: Текущий шаг, исправленные файлы, обнаруженные риски, план.
2. `mapping.json`: Карта соответствия путей, обновляется каждый раз при старте задания
3. `memory.md`: Справочная информация для выполнения заданий или сами задания
4. Перед каждым ответом перечитай ./.refactoring/* и подтверди обновление своего состояния. в начале своего ответа [Прочитана карта состояния].
5. Перед началом своей работы обнови карту соответствия путей записывая в нее свои планируемые действия.
6. По завершению ОБЯЗАТЕЛЬНО обновляй текущее состояние в `state.md` и затем проинформируй меня в конце своего сообщения [Обновлено текущее состояние].
7. Прочитай если есть `memory.md`
   1. Проанализируй документ и составь список задач / tasks для дальнейшего выполнения и положи их в `state.md`
   2. Переодически читай `memory.md` (раз в 30 секунд) на предмет обновления заданий или описания и оповещай меня об этом в начале сообщения [Сверился с записной книжкой]
